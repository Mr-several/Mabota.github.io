使用hexo搭建博客时出现的种种问题(The problem which brought when using hexo to build my blog)

Posted on 2020-06-24

使用hexo搭建博客时出现的种种问题和搭建博客时候的一些基本操作(The problem which brought when using hexo to build my blog and some basic statements)

鉴于个人之前使用hexo搭建过一个博客所以这次搭建的时候也就比较随意很多时候没有按照教程来，所以出现了很多问题。

  1. 第一个出现的问题就是hexo安装不上，之前有一个教程上说安装hexo使用的时间就是比较长，所以也觉得很正常，但是过了好久发现进度条基本完全不动所以开始寻找解决的方法，后来在csdn上找到了解决方法：文章🔗。使用的方法是重新调整npm的安装，可能是版本的问题吧。
  2. 之后出现的问题是，每次使用deploy的时候都要输入密码，超级烦人，但是我其实已经使用SSH key和我的github进行绑定了之后查询到的原因可能由两个，
    • 首先查询到的问题是即使上传了SSH-key但是调用仓库的时候使用的是https的方法进行克隆或者调用的所以这个时候需要使用使用ssh的方法克隆仓库，之后发现在主目录下的config.yml文件中的链接尾部加上,git就可以了，写到这里想起来刚开始在config.yml文件中添加链接应该使用仓库的链接而不是博客的链接，这一点刚开始一直没有注意到。
    • 之后发现还是有时候需要填写密码和账号，不过只是偶尔会出现这样的问题。之后查询可能是由于Github禁用了某个凭证，所以需要更新windows的git凭证管理，fatal:httprequestexception encountered解决方案
  3. 最后出现的问题是上传博客之后博客界面始终不能加载出CSS文件来，也就是没有网站的框架导致整个页面一片空白，之后在CSDN的一个博客里找到了解决方法，解决hexo无法加载css文件的方案特别注意的是其中的的blog位置应该是自己仓库的名字往往是XXX.io，再就是前后的斜杠都不能省略。

搭建博客整整用了一晚上加一早上的时候，过程中还出现和很多小的bug这里就不记录了，总之总算完成了从无到有的这一步。至此自己的博客算是搭建起来了，但是很多东西还没有调整，之后的问题可以一步步解决一步步调整。自己前一段时间的学习效率实在是不高，浪费了很多时间完全赶不上进度，搭建博客的初衷是能够在短期内记录自己的一些进步和学习的感悟，能够实时的给自己的学习增加一点反馈。希望不管在什么时候都能牢记自己的初心踏实走好每一步。
